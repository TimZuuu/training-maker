# Produkty i granice

## Produkt 1 - Master Prompt (Training Master)
Rola:
- prowadzi rozmowę projektową z autorem szkolenia,
- zbiera parametry i dopina specyfikację,
- produkuje finalny prompt dopiero po akceptacji.

Nie robi:
- nie zastępuje blueprintu merytorycznego,
- nie prowadzi końcowego szkolenia z użytkownikiem.

## Produkt 2 - Prompt szkoleniowy PQ (mechanika)
Rola:
- prowadzi użytkownika przez szkolenie,
- obsługuje ustawienia trybu, poziomu, tempa, weryfikacji,
- egzekwuje checkpointy i zbiera log przebiegu,
- obsługuje tryb ADMIN raportowy.

Nie robi:
- nie przechowuje całej wiedzy eksperckiej "w prozie" jeśli ta jest już w BP.

## Produkt 3 - Blueprint PQ (merytoryka)
Rola:
- zawiera wiedzę i przebieg techniczny szkolenia,
- definiuje teksty, zadania, oczekiwane obserwacje i dopowiedzenia,
- jest wzorem do standardu blueprintów.

Nie robi:
- nie jest pełnym promptem systemowym,
- nie niesie całej polityki trybów/admina/ustawień globalnych.

## Granica praktyczna
- BP: "co i po co uczymy"
- Prompt szkoleniowy: "jak prowadzimy interakcję"
- Master: "jak produkujemy prompt szkoleniowy i dbamy o standard"
